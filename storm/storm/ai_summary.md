HOPE is a weekend demo app for a smart rehabilitation glove built on an ESP32. You're the developer, your partner is a med student who built the glove and has a Python script that analyzes the sensor data. The app has two modes — patient and practitioner — and serves a single hardcoded user. No auth, no multi-user, no Bluetooth.
How it works: The ESP32 glove connects over WiFi to an AWS backend (API Gateway + Lambda + DynamoDB + S3). The glove itself is dumb — it streams the same sensor data both times it's used. The app and backend give that data meaning based on when it happens in the flow.
Patient mode is six steps. The patient optionally fills out a short questionnaire (simple form, answers stored in DynamoDB). Then they tap "Assess," flex their hand, and the glove streams sensor data to Lambda, which runs the med student's Python script and returns assessment results plus recommended exercises. The app displays those results. The patient then does the exercises with the glove on — during this the app shows a waiting screen and they can optionally film themselves. When they tap "Done," the backend processes the exercise sensor data through the same script and returns exercise results. Session complete.
Practitioner mode is read-only. They see a list of past sessions and can tap into any one to view the assessment results, exercise results, questionnaire answers, and the video if one was uploaded. That's it — no editing, no plan building, no telehealth.
Backend is minimal: one DynamoDB table with a single session record holding the questionnaire JSON, assessment and exercise results (both from the Python script), S3 keys for raw sensor dumps and video, and a status field. Seven API endpoints covering session creation, questionnaire storage, assess, exercise, video upload (presigned S3 URL), and two GET endpoints for listing and viewing sessions.
What's explicitly out of scope: exercise plan builders, telehealth/video calls, AI/ML on the client, multi-patient management, real authentication, and anything from the original PDF diagram that adds complexity beyond this flow. If it's not in the scope doc, it's not in the demo.
