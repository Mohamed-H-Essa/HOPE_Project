# Localization (i18n / RTL)

The HOPE Flutter app supports **English (`en`)** and **Arabic (`ar`)**. Arabic is right-to-left; the framework handles layout mirroring automatically via `MaterialApp.locale`.

## Architecture

```
LocaleProvider (ChangeNotifier)  ──▶  MaterialApp.locale
         │                                    │
         └─ persisted via                     └─ drives AppLocalizations.of(context)
            shared_preferences                   and Directionality (RTL for ar)
```

- **`lib/l10n/app_en.arb` / `app_ar.arb`** — source of truth for strings (ARB = JSON with ICU placeholders).
- **`lib/l10n/gen/`** — generated by `flutter gen-l10n`. Do not edit by hand. Committed so fresh clones work without a build step.
- **`lib/state/locale_provider.dart`** — holds current `Locale`, persists to `SharedPreferences` key `app_locale`, exposes `setLocale()` / `toggle()`.
- **`lib/widgets/language_toggle.dart`** — two variants:
  - `LanguageToggle` — designed for `AppBar.actions`. Every screen with an AppBar adds `actions: const [LanguageToggle()]`.
  - `FloatingLanguageToggle` — pill-shaped overlay used on AppBar-less screens (welcome screen) via `Positioned.directional` in the screen's `Stack`.
- **`l10n.yaml`** — gen-l10n config (arb dir, output path, class name).

## How to add a new string

1. Add the key + English text to `lib/l10n/app_en.arb`.
2. Add the same key with the Arabic translation to `lib/l10n/app_ar.arb`.
3. Run `flutter gen-l10n` (or just `flutter run` — it regenerates on hot restart).
4. Use it in a widget:
   ```dart
   import '../l10n/gen/app_localizations.dart';
   // inside build():
   final t = AppLocalizations.of(context);
   Text(t.myNewKey);
   ```
5. For dynamic values, use ICU placeholders:
   ```json
   "functionsPassed": "{passed}/{total} functions passed",
   "@functionsPassed": {
     "placeholders": {
       "passed": {"type": "int"},
       "total": {"type": "int"}
     }
   }
   ```
   Call as `t.functionsPassed(3, 5)`.

## How to add a new locale (e.g. French)

1. Create `lib/l10n/app_fr.arb` with all keys translated.
2. Add `Locale('fr')` to `LocaleProvider.supportedLocales`.
3. Update `LanguageToggle` — with more than 2 languages, swap the toggle for a menu (`PopupMenuButton`) listing each locale. Keep the same overlay placement in `main.dart`.
4. Run `flutter gen-l10n`.

## RTL guidelines

Flutter mirrors layout automatically when `Directionality` is RTL, but only if you use directional primitives:

- `EdgeInsetsDirectional.only(start:, end:)` — **not** `EdgeInsets.only(left:, right:)`.
- `AlignmentDirectional.centerStart` — **not** `Alignment.centerLeft`.
- `Positioned.directional(start:, end:)` — **not** `Positioned(left:, right:)`.
- For one-off icons that imply direction (e.g. `chevron_right`), swap based on `Directionality.of(context) == TextDirection.rtl`.
- `Row` mirrors automatically. `TextAlign.start` / `.end` follow locale; avoid `TextAlign.left` / `.right` unless intentional.

Text inside widgets (e.g. `Text(t.someKey)`) picks up direction from ambient `Directionality` — no manual work needed.

## Switching language

Anywhere in the tree:
```dart
context.read<LocaleProvider>().toggle();            // swap en ⇄ ar
context.read<LocaleProvider>().setLocale(Locale('ar'));
```

The change is instant (`notifyListeners()` → `MaterialApp` rebuilds with new `locale`) and persists on next launch.

## Adding the toggle to a new screen

- Screens with an `AppBar`: add `actions: const [LanguageToggle()]`.
- Screens without an `AppBar`: wrap the body in a `Stack` and include `Positioned.directional(top: 8, end: 12, child: const FloatingLanguageToggle())`.

## Testing checklist

When adding a new screen or feature:

- [ ] No hardcoded user-facing strings — all go through `AppLocalizations`.
- [ ] Language toggle is present (AppBar action or floating variant).
- [ ] Both `app_en.arb` and `app_ar.arb` have the new keys.
- [ ] Padding / alignment use the `*Directional` variants.
- [ ] Launch the app, tap the language toggle, navigate to the screen — text translates, layout mirrors, no overflow.
- [ ] Kill and relaunch in Arabic — the locale persists.
- [ ] Run `flutter analyze` — no warnings.
