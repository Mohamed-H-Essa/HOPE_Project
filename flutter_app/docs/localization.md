# Localization (i18n / RTL)

The HOPE Flutter app supports **English (`en`)** and **Arabic (`ar`)**. Arabic is right-to-left; the framework handles layout mirroring automatically via `MaterialApp.locale`.

## Architecture

```
LocaleProvider (ChangeNotifier)  ‚îÄ‚îÄ‚ñ∂  MaterialApp.locale
         ‚îÇ                                    ‚îÇ
         ‚îî‚îÄ persisted via                     ‚îî‚îÄ drives AppLocalizations.of(context)
            shared_preferences                   and Directionality (RTL for ar)
```

- **`lib/l10n/app_en.arb` / `app_ar.arb`** ‚Äî source of truth for strings (ARB = JSON with ICU placeholders).
- **`lib/l10n/gen/`** ‚Äî generated by `flutter gen-l10n`. Do not edit by hand. Committed so fresh clones work without a build step.
- **`lib/state/locale_provider.dart`** ‚Äî holds current `Locale`, persists to `SharedPreferences` key `app_locale`, exposes `setLocale()` / `toggle()`.
- **`lib/widgets/language_toggle.dart`** ‚Äî subtle pill button (globe icon + "EN"/"ÿπ"). Overlaid globally from `MaterialApp.builder` in `main.dart`, so it appears on every screen top-end without per-screen wiring.
- **`l10n.yaml`** ‚Äî gen-l10n config (arb dir, output path, class name).

## How to add a new string

1. Add the key + English text to `lib/l10n/app_en.arb`.
2. Add the same key with the Arabic translation to `lib/l10n/app_ar.arb`.
3. Run `flutter gen-l10n` (or just `flutter run` ‚Äî it regenerates on hot restart).
4. Use it in a widget:
   ```dart
   import '../l10n/gen/app_localizations.dart';
   // inside build():
   final t = AppLocalizations.of(context);
   Text(t.myNewKey);
   ```
5. For dynamic values, use ICU placeholders:
   ```json
   "functionsPassed": "{passed}/{total} functions passed",
   "@functionsPassed": {
     "placeholders": {
       "passed": {"type": "int"},
       "total": {"type": "int"}
     }
   }
   ```
   Call as `t.functionsPassed(3, 5)`.

## How to add a new locale (e.g. French)

1. Create `lib/l10n/app_fr.arb` with all keys translated.
2. Add `Locale('fr')` to `LocaleProvider.supportedLocales`.
3. Update `LanguageToggle` ‚Äî with more than 2 languages, swap the toggle for a menu (`PopupMenuButton`) listing each locale. Keep the same overlay placement in `main.dart`.
4. Run `flutter gen-l10n`.

## RTL guidelines

Flutter mirrors layout automatically when `Directionality` is RTL, but only if you use directional primitives:

- `EdgeInsetsDirectional.only(start:, end:)` ‚Äî **not** `EdgeInsets.only(left:, right:)`.
- `AlignmentDirectional.centerStart` ‚Äî **not** `Alignment.centerLeft`.
- `Positioned.directional(start:, end:)` ‚Äî **not** `Positioned(left:, right:)`.
- For one-off icons that imply direction (e.g. `chevron_right`), swap based on `Directionality.of(context) == TextDirection.rtl`.
- `Row` mirrors automatically. `TextAlign.start` / `.end` follow locale; avoid `TextAlign.left` / `.right` unless intentional.

Text inside widgets (e.g. `Text(t.someKey)`) picks up direction from ambient `Directionality` ‚Äî no manual work needed.

## Switching language

Anywhere in the tree:
```dart
context.read<LocaleProvider>().toggle();            // swap en ‚áÑ ar
context.read<LocaleProvider>().setLocale(Locale('ar'));
```

The change is instant (`notifyListeners()` ‚Üí `MaterialApp` rebuilds with new `locale`) and persists on next launch.

## Testing checklist

When adding a new screen or feature:

- [ ] No hardcoded user-facing strings ‚Äî all go through `AppLocalizations`.
- [ ] Both `app_en.arb` and `app_ar.arb` have the new keys.
- [ ] Padding / alignment use the `*Directional` variants.
- [ ] Launch the app, tap the language toggle, navigate to the screen ‚Äî text translates, layout mirrors, no overflow.
- [ ] Kill and relaunch in Arabic ‚Äî the locale persists.
- [ ] Run `flutter analyze` ‚Äî no warnings.
